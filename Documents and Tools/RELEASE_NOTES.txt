Caelum V3 / Integrated Dashboard V4 Release Notes
=================================================

Release status: public GitHub repository update
Repository: DavidRichardson02/Offline_Flight_Analysis_Dashboard
Primary validation entry point: validate_caelum_release.m

Overview
--------

This release packages the current Caelum offline flight-analysis environment for public repository handoff. It contains the canonical V3 MATLAB analysis/refactor package together with the integrated V4 dashboard surface, firmware telemetry contracts, serial/live playback utilities, representative validation fixtures, and release-facing validation scripts.

The project is no longer only a basic post-flight plotting package. It now acts as a reviewable engineering environment for telemetry ingestion, firmware schema alignment, estimator replay, mission scoring context, dashboard evidence review, live serial telemetry buffering, and validation-driven handoff.

Major Capabilities
------------------

1. End-to-end MATLAB analysis

   - caelum.analyzeLog is the main orchestration entry point.
   - Import is attempted through the strict CSV importer and then through a robust fallback path when needed.
   - Imported data is aligned into the Caelum dashboard contract before cleaning and event detection.
   - Results include cleaned data, detected events, replay outputs, attitude diagnostics, truth metrics, consistency metrics, optional 3D/GPS replay, wind estimates, dashboards, and export metadata.

2. Vertical estimator replay

   - Offline vertical replay supports innovation, NIS, barometric gating, covariance, acceleration-bias, beta-learning, and replay contract checks.
   - Vertical replay validation is covered by validate_vertical_replay_stack.m and replay contract fixture checks.

3. Attitude, 3D/GPS, and wind analysis

   - The default configuration enables attitude replay and 3D/GPS replay when supporting fields are available.
   - The analysis path can attach attitude fields, run the 3D EKF, and estimate wind from the 3D replay output.
   - Focused diagnostics include attitude/gravity provenance and 3D trajectory/wind uncertainty views.

4. Integrated dashboard surface

   - plotDashboard creates the current Caelum Integrated Dashboard V4 surface.
   - The dashboard includes primary flight-state plots, estimator plots, GPS/3D plots, wind plots, optional evidence strips, policy/phase diagnostics, provenance panels, telemetry freshness, and summary information.
   - Optional widgets are controlled through configuration fields such as dashboardWidgets, dashboard.widgets, and layout flags.

5. Firmware/dashboard contract alignment

   - firmware_sdlog_schema.csv mirrors the latest checked SD logger telemetry order.
   - firmware_serial_telemetry_schema.csv mirrors the latest checked serial HDR/TLM payload order.
   - validate_firmware_dashboard_alignment.m treats the firmware reference as the source of truth for dashboard import contracts.
   - The validation checks schema order, field counts, fixture prefixes, GPS extras, serial aliases, live-buffer snapshots, target semantics, and firmware/mission target separation.

6. Serial telemetry and live playback

   - caelum.importSerialTelemetry parses firmware HDR/TLM serial captures without shifting malformed rows into incorrect columns.
   - Live support includes createLiveTelemetryBuffer, appendLiveTelemetryBuffer, snapshotLiveTelemetryBuffer, readLiveSerialTelemetry, and startLiveSerialDashboard.
   - The live playback design supports offline SD replay, captured serial replay, bounded serial buffering, parser-health counters, dashboard snapshots, and true live serialport-backed ingestion.

7. IREC mission profile support

   - caelum.irecMissionProfile defines the IREC 10,000 ft AGL mission scoring profile.
   - The default mission target is encoded as 3048.0 m.
   - Firmware policy target fields remain distinct from the mission/scoring target even when the numeric target values are deliberately aligned.

Release Validation
------------------

Run the release-facing validation gate from the repository root in MATLAB:

    addpath(genpath(pwd));
    validation = validate_caelum_release;

The release validator performs the following top-level checks:

- renders release dashboard windows when MakeDashboards is enabled
- runs validate_firmware_dashboard_alignment
- runs validate_vertical_replay_stack
- runs validate_irec_mission_profile
- runs validate_live_telemetry_import
- combines those results into validation.overallPassed
- throws an error if a required release gate fails

Focused validators are also included for subsystem-level review:

- validate_replay_contract_diff_viewer
- validate_flight_evidence_navigator
- validate_telemetry_freshness_heatmap
- validate_causality_graph
- validate_phase_state_machine_timeline
- validate_policy_decision_audit
- validate_estimator_trust_dashboard
- validate_attitude_gravity_provenance_view
- validate_3d_trajectory_wind_uncertainty_tube
- validate_apogee_sensitivity_waterfall
- validate_monte_carlo_mission_envelope_board

Ignored Outputs

- generated validation outputs

- local IDE metadata
- OS metadata
- local environment files, credentials, keys, tokens, and certificates
